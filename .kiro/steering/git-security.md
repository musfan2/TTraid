# Правила безопасности Git и конфигурационных папок

## СТРОГИЙ ЗАПРЕТ на изменение Git

**КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО:**
- Выполнять `git commit` в любой форме
- Выполнять `git push`, `git pull`, `git merge`, `git rebase`
- Выполнять `git checkout` для переключения веток
- Выполнять `git reset`, `git revert` для изменения истории
- Изменять файлы в папке `.git/`
- Создавать или удалять ветки (`git branch -d`, `git branch -D`)
- Выполнять любые команды, изменяющие состояние репозитория

**РАЗРЕШЕНО только для просмотра:**
- `git status` — просмотр состояния
- `git log` — просмотр истории
- `git diff` — просмотр изменений
- `git branch` — просмотр списка веток (без флагов удаления)
- `git show` — просмотр коммитов
- `git blame` — просмотр авторства строк

## Причина ограничений

Эти ограничения введены для:
1. Предотвращения случайных коммитов с незавершёнными изменениями
2. Защиты истории репозитория от непреднамеренных изменений
3. Сохранения настроек IDE пользователя
4. Обеспечения контроля версий только через ручные действия разработчика
